3. Payments & family invoices

One admin surface: Finance → Family invoices. Parents see shared invoices in the hub.

Create INV-P in Portal → PDF + share row → Parent hub → Tide / Stripe / credit → Paid → Optional Xero payment

Create (admin)

  • Create invoice in Portal — VAT 20% or Exempt
  • Auto number INV-P-####
  • Or upload a Xero/office PDF
  • Share / Hide / Mark paid

Pay (parent)

  • Bank transfer (Tide details) + “I’ve paid”
  • Card / Apple Pay (Stripe Checkout, fee gross-up)
  • Apply open family credit
  • Inline PDF preview before download

Xero (bookkeeping)

  • Push to Xero — creates ACCREC, stamps GUID
  • Export CSV — fallback import
  • When paid in Portal → write-back POST /Payments if GUID present

What is NOT an admin screen

  • Tide credentials (secrets only)
  • Stripe keys (secrets + webhook)
  • Old “Xero / finance” builder → redirects here

4. Re-enrolment 2026/27

Parents confirm slots + funding/payment plan. On first submit (parent-pays), Portal auto-creates the INV-P instalment schedule.

Lookup child → Keep / change / withdraw slots → Choose pay plan → Submit → INV-P schedule

Payment plans (parent-pays)

  • Bank transfer — term / flexi / year (no admin fee if on time)
  • GoCardless — monthly-by-term · £1.50 / instalment
  • Own arrangement — only if they cannot meet fixed dates · £50 / term + 2-session prepaid buffer
  • auto_continue on year-long standard plans

Funder-paid (LA / NHS invoice)

  • No parent payment schedule
  • No auto INV-P on submit
  • Office invoices the funder separately

Resubmits do not recreate invoices — office adjusts in Family invoices.

5. Prepaid buffer & payment holds

Own-arrangement families must keep ~2 sessions prepaid per service. Pressure is staged; hard cut stays manual.

Nightly buffer check → Soft hold → Remind → Hold 1 session → Pay unlocks
  • Available (v1): open credits − overdue unpaid invoices
  • Cron: ~06:00 London · auto soft-hold / clear soft-hold only
  • Never auto: hold session or hard cut
  • Admin chips: Buffer low · Soft hold · Session held

7. Absent → makeup → credit

Parent reports absence from the hub; admin validates; outcomes feed credits / makeups / roster.

Report absent → Proof if Unwell → Admin decide → Makeup offer / credit / noted
  • Date chips show absent / cancelled / done / next
  • Makeup grant → parent accepts venue/slot → roster MakeUp
  • Decline can forfeit; club cancel early can create credit/refund path
  • Credits appear in family ledger and can pay an invoice

8. Course funding (L&D)

Staff learning & development funding applications — separate from family invoices. Lives in admin L&D review + letter PDFs.

What it is

  • Staff apply for course funding
  • Admin reviews / decides (Approved / Declined)
  • Decision letter PDF saved to applicant documents

Linked to

  • portal_ld_funding_* + letter helpers
  • documents (training letters)
  • Not Family invoices / parent hub money

If someone says “funding” in a parent context they usually mean LA/NHS / Direct Payments on re-enrol — different from staff Course funding (L&D).

Quick glossary

INV-P

Portal-generated family tax invoice number series (separate from Xero INV-).

Own arrangement

Flexible pay timing + £50/term fee + must keep 2 sessions prepaid.

Funder invoice

LA/NHS pays the club — parent does not get a payment plan.

Direct Payments

Parent receives LA money and pays the club (often GoCardless / bank).
