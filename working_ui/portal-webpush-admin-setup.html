Lo viste en supabase secrets list con nombre + DIGEST (hash). Eso es correcto. No hace falta volver a pegar el listado en chats.
Seguridad: No compartas en público el valor de PORTAL_PUSH_WEBHOOK_SECRET, SUPABASE_SERVICE_ROLE_KEY ni VAPID_PRIVATE_KEY. En los webhooks solo lo copias del Dashboard de Supabase a mano.

Qué falta para que las alertas admin funcionen:

  1. SQL de la tabla dedupe (una vez).
  2. Edge Function desplegada (si no está en remoto).
  3. Cuatro webhooks en el Dashboard (este documento).
  4. Activar notificaciones en admin_dashboard.html.

1 SQL — tabla dedupe

En Supabase → SQL Editor (proyecto Portal), ejecuta el archivo del repo:

supabase/migrations/20260617120000_portal_webpush_admin_alert_sent.sql

Crea la tabla portal_webpush_admin_alert_sent para no enviar el mismo push dos veces.

2 Edge Function — desplegar

En PowerShell, carpeta del repo, con CLI logueado y proyecto Portal enlazado:

cd C:\Users\info\PORTAL
supabase functions deploy portal-push-dispatch-admin-alert

Comprueba en Dashboard → Edge Functions que aparece portal-push-dispatch-admin-alert como activa.

3 Webhooks — lo importante (4 veces)

Abre el enlace directo a webhooks del proyecto Portal:

Dashboard → Integrations → Webhooks

La ruta antigua /database/hooks da 404. Usa Integrations → Webhooks.

Valor del header (una sola vez)

Antes de crear webhooks, copia el secret desde:

Project Settings → Edge Functions → Secrets (o donde guardaste el valor al crearlo):

  • Nombre del header: x-portal-webhook-secret
  • Valor: el contenido de PORTAL_PUSH_WEBHOOK_SECRET (no el DIGEST del CLI)

4 Probar

  1. Dashboard → Edge Functions → portal-push-dispatch-admin-alert → Logs.
  2. Genera un evento real o de prueba (p. ej. late submission pending, o mensaje DM de un staff que no sea admin/ceo).
  3. En logs debe aparecer actividad y respuesta 200. Si ves 403, el header del webhook no coincide con el secret.

5 Admin dashboard — activar push en el navegador

  1. Entra como admin/CEO en admin_dashboard.html.
  2. Activa notificaciones (campana / ajustes de push) hasta ver texto tipo “On — including alerts when the app is closed”.
  3. Al pulsar una notificación admin debe abrir el admin con alertas (?portalOpen=alerts), gracias a PORTAL_PUSH_ADMIN_OPEN_URL.

Resumen rápido

  • Staff sigue usando PORTAL_PUSH_OPEN_URL → staff_dashboard.
  • Admin/CEO usa PORTAL_PUSH_ADMIN_OPEN_URL → admin_dashboard (ya configurado).
  • Los 4 webhooks llaman a la misma URL de Edge Function.
  • Chat: no notifica si quien escribe ya es admin/ceo.
