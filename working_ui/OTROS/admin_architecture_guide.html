0. Scope and agreed rules

This document covers the admin only (working_ui/admin_dashboard.html and what it connects to: roster, Supabase overrides, booking localStorage, etc.).

Do not change staff_dashboard.html or staff_dashboard.html unless explicitly requested later — those dashboards are considered stable and correct.

Parents do not use the internal portal: communication is via WhatsApp, email, occasional public HTML, and ClassForKids where applicable.

1. Four pillars (mental model) + fifth “System”

1.1 Pillar A — Clients participant + contract + billing

  • Covers: participant, LA/private context, intake, lifecycle; payments (due, partial, balance, due dates); parent notices about charges; absences/compensation when linked to policy.
  • Does not cover: staff compliance (DBS, etc.) → Pillar B; master product catalogue → Pillar C (link only).

1.2 Pillar B — Instructors people ops

  • Covers: staff record, declared availability, onboarding, interviews, policies; Staff Matrix-style grid (Excel today).
  • Does not cover: “who is in the pool at 4:30 pm today” → roster + overrides (Pillar D).

1.3 Pillar C — Services / offer

  • Covers: programmes, venue, price per session, sessions per term; re-enrol rules (public form as business reference).
  • Does not cover: assigned day slots → D; ledger → A.

1.4 Pillar D — Operational cross-cut

  • Covers: day/week from roster; Supabase overrides; aggregated capacity; parent messages for instructor changes (draft).
  • Does not cover: full term financial contract → mentally Pillar A.

1.5 System: integrations, global exports, settings, logout, placeholders.

2. Suggested navigation map (documentation)

Grouping the sidebar into pillar-aligned blocks avoids mixing responsibilities.

  • Overview: Dashboard; Operator (demo shortcuts).
  • Clients & revenue (A): Bookings; Invoices; (future) single participant record.
  • Services & catalogue (C): (future) dedicated module; catalogue in web/Excel today.
  • Schedule & delivery (D): Scheduling & Covers; (optional) Operations live board (demo today).
  • People — staff (B): Staff & HR; Onboarding; Policy sign-offs.
  • Comms & quality: Communications; Reports & incidents.
  • System: Settings.

4. Data flows (who feeds whom) 
[Spreadsheet / STAFF_DASHBOARD_SOURCE + Adapter]
  → sessions by weekday + slots by date (Pillar D)
  → same base for Bookings and Scheduling

[Supabase schedule_overrides]
  → day changes on anchors (staff, client, time, venue, date)
  → “effective” participant in UI → consistent with Bookings

[localStorage portal_booking_meta_v1_* / portal_booking_finance_v1_*]
  → intake, funding, parent contact, payment plans, transactions (Pillar A)
  → evolution: migrate to Supabase

[Commercial Excel / re-enrol web form]
  → today: operational truth + ClassForKids
  → future: term contract + catalogue (C → A)

[Excel Staff Matrix / Job applications]
  → Pillar B specification; does not feed Staff/Lead by design
5. Evolution phases (without touching Staff/Lead)
  • Phase 1 (now): D + A on roster + LS + overrides + parent notices.
  • Phase 2: persist A (and C) in DB; client record / billing matrix.
  • Phase 3: HR module in admin (import Staff Matrix).
  • Phase 4: services catalogue feeding prices / internal re-enrol.
6. External references (business model)
  • Public re-enrol (parents): clubsensational.org/RE-ENROLMENTS/ — services by day, price per session, number of term sessions, funding, payment method, total. Parents do not pick times: they keep slots from the previous term; cost varies with more/fewer sessions.
  • ClassForKids: family-facing shop window; heavy operations in spreadsheets + admin portal.
7. Open questions — inferred answers
  1. Re-enrol URL: provided; active (e.g. Summer 2026) on the website.
  2. First large Excel sheet: term operational workbook; several rows per client/months/blocks.
  3. Second sheet: parallel view oriented to billing/LA/invoices.
  4. Two Bookings tabs vs single record: two tabs today; single record would be a UX improvement.
  5. CFK “Updated”: reasonable per line or block; optional sync per term.
  6. Monthly matrix vs list: Excel habit = matrix; fast implementation = list + CSV.
  7. Assign participant from Manage/drawer: correct today; modal only in Bookings = pending if requested.
  8. After weekly capacity, “schedule of the day” list: next logical block in Bookings or Operations.
8. HR summary (Excel) vs portal — without duplicating Staff/Lead

Job Application Form (Responses): full application pipeline (RTW, education, employment, DBS declaration, references…). No persisted equivalent in Staff/Lead; fits Admin Pillar B (recruitment).

Staff Matrix: Employees info, Employee Docs (DBS, first aid, STA…), Induction & Safeguarding, Shadowings, Observations, Interviews, Paid courses, Job application (bank details — highly sensitive), Health questionnaire.

Staff/Lead today: external training and session identity; do not store HR expiry dates or matrix. That is future Pillar B in Admin (or external HR with API).

10. Recent portal features (production, Jun 2026)

Summary of what shipped on the Portal static app and Supabase backend — for admin operators and support.

  • Admin topbar — live presence: one row of first names only (no Admin / Staff / Onboarding columns). Link Guide opens this page in English.
  • Admin hub — Operator: category colour is pink (sidebar + dashboard tiles).
  • Admin hub — Documents: eight compact icon tiles in one row (Payslips, Timesheets, Expenses, PINs, Certificates, Passports, Checklists, First aid).
  • Payslips: admin uploads monthly PDFs per worker (portal_payslips); workers see them under My Documents → Payslips. Migration: 20260610140000_portal_payslips_documents.sql.
  • Participant achievements: staff/lead take in-app photos per participant (not saved to phone gallery). Up to 10 drafts per participant per day; attach up to 3 when submitting session feedback. Admin views under Participant achievements.
  • Session feedback — voice: mic on Positive feedback and Relevant information. Browser Web Speech when available (live text); Spanish and Italian auto-detected from staff name → live speech with translation to English for the record. Whisper edge function is fallback only when Web Speech is unavailable.
  • Session feedback — photos: optional checkboxes for achievement drafts; finalize runs via RPC portal_finalize_achievement_photos (requires Supabase auth — PIN-only submit saves feedback but may skip attach if not signed in).
  • Day Centre — lead feedback window: 30 minutes before session end (was 15).
  • Announcements: production cleanup migration keeps one portal-ready announcement; acks table portal_staff_announcement_acks. Migrations: 20260610120000, 20260610150000.
  • Mobile screenshot guard: pauses black overlay during camera/mic capture so feedback and achievements do not freeze on a black screen.
