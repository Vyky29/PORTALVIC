GRID MADRE (recomendado — como la hoja del club)
------------------------------------------------
  Abre directamente en el navegador (doble clic, sin servidor):

    working_ui/roster_term_master_review.html

  Vista por día, columnas instructor + venue + servicio, filas de 30 min.
  Edita participante, hora y nota de pool (Big Pool, Lane (DE), Hub Room…).
  Leyenda de colores por tipo de servicio.

  Botones:
    • Guardar borrador — localStorage en este navegador
    • Exportar MADRE — descarga roster_term_master.json
    • Reset — vuelve al seed original

  Tras exportar MADRE, copia el JSON a working_ui/portal/roster_term_master.json
  y en terminal:

    python database/roster_review/apply_madre_after_manual_edit.py

  (Incluye sync bundle, seed, boot y staff-shifts.csv desde MADRE.)
  Luego commit + push → Vercel.

  Si cambias roster_term_master_seed.json, regenera el boot (para file://):

    python database/roster_review/build_roster_term_master_seed_boot.py

ARCHIVO PARA EDITAR (Excel / Google Sheets / Numbers)
-----------------------------------------------------
  database/roster_review/participants-by-day-area-notes.csv
  database/roster_review/staff-shifts.csv

  1321 filas exportadas del roster que usa el dashboard hoy
  (working_ui/portal/staff_dashboard_spreadsheet_bundle.js).

COLUMNAS
--------
  date        → YYYY-MM-DD (vacío = plantilla recurrente sin fecha)
  weekday     → Monday … Sunday
  client      → nombre del participante
  service     → Aquatic Activity, Multi-Activity, Day Centre, etc.
  time_slot   → ej. 2 to 2.30, 4.30 to 5, 9 to 9.30
  instructor  → ej. JAVIER, ROBERTO, LULIA, MICHELLE
  venue       → Acton, SwimFarm, Northolt, Westway
  notes       → ÁREA que sale en el dashboard (icono + texto)

  La columna que normalmente corriges es **notes**.

VALORES VÁLIDOS DE notes (área)
-------------------------------
  Teaching Pool
  Small Pool
  Big Pool
  Hub Room
  Room 2
  Lane (SE)
  Lane (DE)
  Wall
  Gym
  Climbing Wall

Reglas que ya acordamos:
  • Domingo SwimFarm, Aquatic Activity, 9 to 9.30  → Small Pool
  • Domingo SwimFarm, Aquatic Activity, 2 to 2.30 / 2.30 to 3 → Big Pool
  • En domingo en SwimFarm NO usar Teaching Pool en esos bloques acuáticos

CÓMO RECARGAR DESPUÉS DE EDITAR
-------------------------------
1. Guarda el CSV (UTF-8, mismas columnas, no borrar la fila de cabecera).

2. En terminal, desde la raíz del repo:

     python database/roster_review/apply_participants_by_day_csv.py

   Eso actualiza el campo area en los bundles que despliega el portal:
     • working_ui/portal/staff_dashboard_spreadsheet_bundle.js  ← Vercel
     • working_ui/portal-shared-js/staff_dashboard_spreadsheet_bundle.js
     • database/staff_dashboard_spreadsheet_bundle.js

3. Pide en Cursor: "aplica el CSV de roster_review y haz push"
   (o commit + push tú mismo).

4. Vercel redeploya; el staff dashboard mostrará las notas nuevas.

QUÉ HERRAMIENTA ADMIN USAR (para que cuadre con staff)
------------------------------------------------------
  El staff dashboard combina tres capas:
    1) Bundle (MADRE / sync)  2) portal_roster_rows  3) schedule_overrides

  | Quiero… | Herramienta | Escribe en… |
  |---------|-------------|-------------|
  | Familia nueva en hueco ABIERTO (trial) | New participant | portal_roster_rows |
  | Cambiar niño/instructor/hora/pool TODO el término | Edit term slot | portal_roster_rows (+ eventos) |
  | Cambio de UN día: absent, cancel, cover, make-up | Schedule & Covers | schedule_overrides |
  | Ver la hoja del club (no guardar) | Spreadsheet reference | (solo lectura) |
  | Cambios masivos del término (como la hoja MADRE) | roster_term_master_review → sync | MADRE → bundle + merges |

  Reglas:
  • No uses solo Schedule & Covers para un cambio de término (ej. Joel→Anas 4 semanas):
    usa Edit term slot o MADRE + sync.
  • New participant ya guarda en Supabase; staff lo ve tras recargar el portal.
  • Tras MADRE/sync: commit + push → Vercel.
  • Si algo no cuadra: revisa portal_roster_rows + schedule_overrides en Supabase.

  Sync MADRE desde terminal:
    python database/roster_review/sync_roster_madre_to_portal.py
